How the Synthetic Learner Works

From a student profile to a structured learning diagnostic — five stages, one iterative content-processing loop, scored against real student data.

One learner × one instruction run — step by step
Step 1  ·  Define Persona
Encode who the learner is
Prior knowledge and misconceptions → belief weights on specific concepts
Learning strategy (scan, deep read, experiment) → behavior probabilities per content type
Goal orientation and ego-protection → persistence thresholds and skip triggers
Values filter → domain-specific acceptance/rejection rules
Output: structured persona profile
Step 2  ·  Queue Instruction
Break content into processable chunks
Text chapter → sentences grouped into semantic chunks (topic shifts, paragraph breaks)
Simulation → discrete slider interactions as separate observation events
Each chunk processed once, in order — no skipping ahead, no backtracking
Output: ordered content queue
Step 3  ·  LLM Decision (↻ repeats per chunk)
Persona + content → per-chunk action
Input: [persona context] + [current chunk]
Outputs: attend or skip  ·  concepts to extract  ·  mechanism label to apply  ·  confidence weight
Same chunk, different persona → different decision. Bob skips mechanism text; Alfredo reads but filters human causation; Sally adds isolated nodes with no links.
Every decision logged to behavioral trace — attend, skip, filter, stop
Output: per-chunk action + trace entry
Step 4  ·  Update Causal Map
Build the knowledge state incrementally
Attended chunks → nodes (concepts) and directed links added to graph
Each link labeled: mechanism ("absorbs infrared"), sign (positive/negative), confidence weight
Skipped or filtered chunks → map unchanged; gaps are as informative as the content added
Map can be queried mid-run ("does my current map support this claim?")
Output: incrementally built causal graph
Step 5  ·  Learning Monitor
Score the map against expert ground truth
Node recall: what % of expert concepts did the agent capture?
Link accuracy: are directions and signs correct?
Chain depth: are multi-step causal paths complete or broken?
Mechanism labels: are links named with mechanism words, or left blank?
Misconception persistence: do wrong links survive the full run?
Output: structured design diagnostic
↻ The core loop: Steps 2 → 3 → 4 repeat for every content chunk in sequence. A 30-paragraph chapter = 30 decision loops. A 10-slider simulation = 10 observation loops. The behavioral trace (every attend / skip / filter decision) is what you see in the Learner Profiles tab below.

Experimental design — what happens in each run
Text + quiz run
The agent reads a written textbook chapter on greenhouse gases. Paragraph by paragraph, it decides — shaped by the persona — which sentences to attend to, which to skip, and what causal triples to extract. It can also use a built-in quiz to test the map it is building. The output is a causal map built from language.
Simulation run
The agent interacts with an interactive science simulation. It varies sliders (CO₂ level, methane, etc.), observes what changes, and adds causal links from what it sees. No text to parse — causation is inferred from observed co-variation between variables. The output is a causal map built from experimentation.
1
Pair one persona with one instruction version
Each run is one combination: e.g. Bob × text chapter. Three personas × two versions = 6 runs total. Each run is independent — the agent starts with only the persona's prior beliefs, no shared memory across runs.
2
The agent engages the content step by step
The persona parameters (reading speed, ego-protection, goal orientation, prior knowledge, persistence) are translated by the LLM into per-step probabilities: attend vs. skip, add a node vs. ignore, stop early vs. continue. The behavioral trace records every decision.
3
Betty's Brain causal map is built incrementally
As the agent reads or experiments, it adds nodes (concepts) and directed links (causal relations) to a structured graph — the same causal map format used in real Betty's Brain studies with human students. Links can carry mechanism labels (e.g. "reflects infrared") and confidence weights.
4
The Learning Monitor scores the result
The agent's map is compared to an expert ground-truth map. The monitor reports: % of expert nodes captured, link accuracy, mechanism label presence, misconceptions that persist. This is the same evaluation metric used in Betty's Brain human studies (agent-to-student correlation r ≈ .6).
Why an agent rather than a human student?
Running a real student through both instruction versions takes hours and requires IRB approval, scheduling, and payment. Running all 6 conditions with real students is a multi-week study — and that's for three personas. A synthetic agent runs each condition in minutes, produces a structured comparable output, and can be re-run immediately on a revised design.

For program teams investing in curriculum quality at scale, this shifts the question from "did our pilot work?" (answered after significant resource commitment) to "which design will work, and for whom?" (answered before resources are committed). The synthetic learner is a design diagnostic, not a replacement for human validation — it front-loads iteration so that human studies evaluate instruction that is already well-calibrated.

Instructional Implications

What the synthetic runs reveal for instructional design — organized by finding, not by student.

→ No single instruction format is sufficient Text-only and simulation each produce systematically different gaps. Text builds some causal structure but misses coverage; simulation adds breadth but loses mechanism precision. The two formats are not interchangeable — they fail in different places.
Teacher actionSequence text AND simulation within the same unit — not as alternatives. Use text to establish mechanism vocabulary and causal chains; use simulation to test and extend pairwise relationships.
Learning outcomeStudents produce maps with both correct multi-step chains and broad coverage — neither format alone achieves both, so neither should be the only format used.
Evidence:Bobtext: 3 endpoint links, no chain · simulation: 4 links, no mechanism labels
→ Domain vocabulary is a prerequisite, not a learning outcome Students cannot build a causal map from content they cannot name. Observing CO₂ and temperature co-vary in the simulation means nothing without the term "infrared absorption" to anchor why. Good reading skills do not substitute for missing domain vocabulary.
Teacher actionTeach mechanism terms — infrared absorption, albedo, radiative forcing — with explicit definitions and concrete examples before introducing causal content that depends on them.
Learning outcomeStudents can label causal links in the map with mechanism words, not just draw unlabeled arrows or note that two variables "go together."
Evidence:Sallynear-empty map in both runs · correlation observed but mechanism unnamed
→ Standard assessments miss the most important failure modes Multiple-choice and accuracy scores make structural and values-based gaps invisible. Alfredo knows the facts — a recall test would not flag his missing human-causation link. Bob's endpoint answers look correct on a quiz, but the causal chain connecting them is absent. The gap is real; the test simply doesn't ask for it.
Teacher actionAdd causal path trace tasks to assessments: ask students to draw the chain from human emissions to surface temperature, name each intermediate step, and label the mechanism on each link.
Learning outcomeAssessment captures chain completeness, mechanism labeling, and whether the student has connected anthropogenic causation — not just whether they can recall gas names or pick correct endpoints.
Evidence:AlfredoBobboth would pass recall test · causal map reveals the gap
→ Self-explanation must be explicitly taught as a strategy No learner naturally pauses to build a causal model unprompted. Bob scans for quiz answers. Sally reads headings. Alfredo reads carefully but filters for worldview fit. Engagement with content is not the same as constructing a mental model — that construction is a learned skill.
Teacher actionBuild self-explanation prompts directly into instruction: "What does this step cause, and why?" "What is the mechanism between X and Y?" Model a completed causal chain before asking students to build their own.
Learning outcomeStudents internalize the habit of connecting nodes with mechanism labels during learning — not just after, as a review task — so the causal map reflects genuine understanding, not post-hoc recall.

Applications

Six Use Cases

The synthetic learner is a GPS for instruction — not one fixed route, but a tool that answers different navigation questions depending on who asks.

🔬

Evaluate Instructional Materials

Run multiple synthetic students through competing versions of content and compare causal map outcomes. Find which version works best — and for whom.

  • Which version produces the most complete causal map?
  • Which leaves the most misconceptions intact?
  • Does one version favour a specific student type?

GPS: "Which route gets the most drivers to the destination?"

🧑‍🏫

Teacher Professional Development

Show teachers a gallery of behavioral traces. Teachers practice recognising student patterns and learn which instructional moves help which learner types.

  • What does a "confident-but-wrong" reader look like?
  • How do you catch Alfredo's worldview conflict early?
  • Which formative checks work for disengaged profiles?

GPS: "Teaching the driver to read road signs."

🗺️

Curriculum Gap Analysis

Identify which causal links are consistently missing across all personas. Those gaps expose weaknesses in the instructional design — not the students.

  • Which concepts are never acquired regardless of persona?
  • Which links require explicit scaffolding to form?
  • Are multi-step chains supported or must students infer them?

GPS: "Finding the roads missing from the map."

🎯

Differentiated Instruction Design

Identify which approach works best for which persona. Design adaptive pathways and set up targeted scaffolding before students encounter the material.

  • Should hands-on learners always start with simulation?
  • Do ego-protective learners need low-stakes quiz framing?
  • How much does prior misconception slow learning?

GPS: "Different routes for different vehicles."

⚡

Pre-Deployment Stress Testing

Test new educational software before human pilots. Get signal fast, revise early, and sharpen human-study hypotheses at a fraction of the cost.

  • Does this simulation break for low-persistence learners?
  • Is the reading level accessible for a reluctant reader?
  • Are quiz questions ambiguous from a misconception lens?

GPS: "Test drive the route before the road trip."

💡

Research Hypothesis Generation

Generate predictions about human learning outcomes from persona × instruction combinations. Use these to design tighter human studies.

  • What effect sizes would self-explanation prompts add?
  • Which student types benefit most from a gameshow format?
  • Would front-of-class display motivate or threaten Priya?

Technical Innovation

AI Infrastructure — Not a One-Size-Fits-All Product

Most ed-tech tools produce a single evaluated curriculum. This system produces infrastructure that gets smarter with every use — a dynamic research repository, not a fixed deliverable.

Traditional approach
✗ One curriculum evaluated per study
✗ Results are static — don't improve with new data
✗ High cost per new design iteration (weeks, IRB, payment)
✗ Domain-locked — new domain = start from scratch
✗ Learner profiles are assumptions, not versioned assets
AI Infrastructure
✓ Any curriculum evaluated, any time — same infrastructure
✓ Predictions improve as calibration data accumulates
✓ Minutes per iteration — design, re-run, compare
✓ Domain-agnostic — swap the expert map, keep everything else
✓ Persona library grows as a versioned, reusable research asset
🧬

Persona-as-Code

Learner profiles are structured, versioned parameters — not fixed character sketches. Each persona encodes prior knowledge, misconception weights, behavior probabilities, and goal orientation as machine-readable parameters.

Research value: Personas calibrated against real cohort data become a growing library of learner models — usable across any future curriculum evaluation.
🗺️

Causal Map as Universal Knowledge Currency

The causal graph is domain-agnostic and machine-readable. Any subject with a causal structure — health behavior, agricultural practice, civic knowledge, financial literacy — can be evaluated with the same infrastructure. Change the domain: update the expert map. Everything else stays the same.

Infrastructure value: One system architecture → multiple program areas.
↻

Compounding Calibration

Every human study that validates a synthetic prediction adds to the calibration dataset. The system's accuracy compounds with use — unlike a static curriculum product that depreciates as student contexts change. The current r ≈ .6 is a starting point, not a ceiling.

Investment logic: early commitment produces durable infrastructure, not a one-time deliverable.
📂

Behavioral Trace as Research Byproduct

Every run produces a full behavioral trace — every attend, skip, filter, and stop decision for every content chunk. This is a research data asset, not a cost. Over time, trace data across personas and domains becomes a repository for studying how different learner types engage with different instruction types.

Longitudinal value: the data compounds across programs — each run adds to the corpus.
🔩

Full Modularity — Swap Any Component

Instruction, persona, and evaluation are independent modules. Change the instruction: the same personas re-evaluate it automatically. Add a new persona: the same instruction is tested against a new learner type. Upgrade the LLM: everything downstream improves. No component is locked to another.

Operational value: teams can extend the system without rebuilding it.
📈

The Repository Grows — It Doesn't Depreciate

A curriculum product is finished when it ships. This system is in its earliest state when it ships. Each grantee program that runs evaluations adds calibration data, new personas, and new domain expert maps — making every subsequent evaluation more accurate.

Portfolio value: each program's use makes every subsequent evaluation more accurate.
